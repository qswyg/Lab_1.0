##  Запуск проєкту
1. `npm install` — встановлення залежностей.
2. `node src/db/initDb.js` — створення бази та індексів.
3. `node src/server.js` — запуск сервера.

##  Схема бази даних
- **Users**: Користувачі (email — унікальне поле).
- **Categories**: Дисципліни 
- **Notes**: Нотатки. Зв'язок `1:N` з Users та Categories.
  - Використано `ON DELETE CASCADE`: при видаленні користувача всі його нотатки видаляються автоматично.
  - Створено індекс `idx_notes_title` для прискорення пошуку за заголовком.

## API Endpoints
- `GET /api/notes` — Отримання всіх нотаток з використанням **JOIN** (підтягує назву категорії).
- `GET /api/notes?categoryId=1` — Фільтрація за предметом.
- `GET /api/notes/stats` — Агрегація (кількість нотаток у кожній категорії).
- `POST /api/users` — Реєстрація (повертає **409**, якщо email зайнятий).
- `DELETE /api/notes/:id` — Видалення нотатки.

## Обробка помилок
Реалізовано централізований Middleware в `server.js`, який ловить усі помилки та повертає їх у форматі JSON із правильним кодом стану (400, 404, 409, 500).
